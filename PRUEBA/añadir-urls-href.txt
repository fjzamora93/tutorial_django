¿CÓMO AÑADIR HREF Y URLS EN DJANGO DE FORMA RELATIVA? (3 fases)


PARA EL CSS
1. CREACIÓN DEL DIRECTORIO STATIC

En primer lugar, dentro de nuetro proyecto debemos crear una carpeta llamada "static".
Este directorio es el que contendrá todos los archivos estáticos que vayamos a usar en nuestra app,
como puede ser el CSS, pero también imágenes.

Como buena práctica, podemos crear un subdirectorio para cada página.
Por ejemplo, para el index o el main, será:
    static/hello

Aunque podemos crear tantos subdirectorios como sea necesario.


PARTE 2: CARGAR EL STATIC 

Antes de comenzar nuestro documento de HTML tenemos que cargar el static.
Este static es esto:

    {% load static %}


PARTE 3: INSERTAR EL CÓDIGO CON REFERENCIA AL STATIC

Una vez está cargado antes de nuestra etiqueta <html:5>, 
ahora podemos añadir nuestro link al CSS, pero en lugar de escribir
la ruta absoluta (que puede cambiar) o la ruta relativa (que no funciona)
vamos a decir que es un estático. Esto lo haremos de esta manera:

    <link rel="stylesheet" href="{% static 'hello/style.css' %}" type="text/css">

Al hacerlo así, estamos diciendo a django que se imagine dónde está static y lo cargue a la url.


PARA URLS QUE LLEVEN A OTRA página

Aunque nuevamente podríamos usar rutas absolutas, lo cuál provocaría 
problemas en caso de cambiar la estructura de la página web. Lo ideal es usar 
rutas relativas, que tienen el siguiente formato:

    <a href="{% url 'index' %}"> Home </a>

    <a href="{% url 'añadir' %}"> Añadir una nueva tarea a la lista </a>



¿Por qué funciona esto? Porque recuerda que tenemos un archivo con todas las URLS,
y es ahi donde deberíamos cambiar eso.
